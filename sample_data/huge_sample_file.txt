--------------------------------------------------
GIT COMMANDS TO RESENT CHANGES
--------------------------------------------------

git reset --soft HEAD~1

What it does: It moves the HEAD back to the previous commit.
Result: Your changes are kept and will be moved back into your "Staged" area (the index). You can then make more changes, re-commit, or edit your files.

git reset HEAD~1

What it does: This is the default reset mode. It keeps the changes in your files but they are unstaged.

git reset --hard HEAD~1

What it does: It moves the HEAD back and forces your working directory and staging area to match that previous state.

Result: Any work you did in that commit is gone.
